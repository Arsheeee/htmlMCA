WHAT IS CSS?

CSS stands for Cascading Style Sheets. Itâ€™s a language used to describe how HTML elements should look on a webpage. While HTML provides the structure (headings, paragraphs, images, etc.), CSS controls the presentation things like:

  • Layout (how elements are arranged on the page)
  • Colors (text color, background color, borders)
  • Fonts (size, family, weight, style)
  • Spacing (margins, padding, line height)
  • Animations & transitions (smooth effects, hover states, keyframes)
For Example: <!DOCTYPE html> <html> <head> <style> body { background-color: #f0f0f0; font-family: Arial, sans-serif; } h1 { color: blue; text-align: center; } p { font-size: 18px; line-height: 1.5; } </style> </head> <body> <h1>Hello, CSS!</h1> <p>This is styled with CSS.</p> </body> </html>

Properties Of CSS

INLINE

  • Written directly in the HTML element using the style attribute.
  • Affects only the specific element where it is applied.
  • Quick to test or apply a single change.
  • Increases HTML file size if used too much.
  • Hard to maintain because styles are mixed with content.
  • Has the highest priority (overrides internal & external CSS).
  • Not recommended for large projects or multiple pages.

INTERNAL

  • Written inside a <style> tag in the <head> section.
  • Applies to the whole page where itâ€™s written.
  • Keeps CSS separate from content (better than inline).
  • Easier to maintain than inline but not reusable across pages.
  • Good for single-page websites or demos.
  • If conflict with external CSS, internal wins (higher priority).
  • Adds extra code inside HTML (less clean than external).

EXTERNAL

  • Written in a separate .css file.
  • Linked to HTML with a <link> tag.
  • Reusable across multiple pages.
  • Keeps HTML clean and improves readability.
  • Best for large projects (easy to update styles).
  • Reduces page size if stylesheet is shared.
  • Has lowest priority (can be overridden by inline or internal).
